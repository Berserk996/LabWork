Тест план
Основные цели тестирования: 
   Что должны протестировать?
   Что будем тестировать?
   Когда будем тестировать?
   Критерии для начала тестирования.
   Критерии для окончания тестирования.

1.Что должны протестировать?
Будем тестировать вэб-приложение интернет-магазин.


2.Что будем тестировать?
список функций и описание тестируемой системы и её компонент в отдельности:
a)Главную страницу
b)Регистрацию и авторизацию
c)Переходы по страницам и страницы товара
d)Тестирование корзины и оформление заказа
e)Тестирование локализации
f)Тестирование нагрузки, стабильности, и объёма
g)Тестирование безопасности
h)Тестирование бизнес-логики
i)Тестирование API запросов
j)Usability тестирование
k)Кроссбраузерное тестирование
l)Мобильное тестирование


3.Когда будем тестировать?
а)После ознакомления с документацией, сбора и анализа требований, кодирования(разработки), тестировщики приступят к этапу тестирования.


4.Критерии для начал тестирования
a)Готовность тестовой платформы
b)Законченность разработки требуемого функционала
c)Наличие всей необходимой документации


5.Критерии для окончания тестирования
a)Требования к количеству открытых багов выполнены
b)Выдержка определенного периода без изменения исходного кода приложения
c)Выдержка определенного периода без открытия новых багов
